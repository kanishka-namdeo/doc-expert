# Documents

## Purpose

Document viewing and management pages.

## Ownership

- Document list page
- Individual document viewing page (`[id]`)
- Document metadata display

## Local Contracts

- Requires authentication
- Document access controlled by permissions system
- Uses document API routes under `/api/documents`

## Work Guidance

- List view shows documents the user has access to
- Detail view renders document content with AI-powered features
- Document permissions determine view/edit access
